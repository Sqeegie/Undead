Certains scripts composant SmartLight comportent des arguments et sont conçus pour être utilisés comme des fonctions. Some component scripts SmartLight include arguments and are designed to be used as features.

SL_light_cast_obj(obj_index,mask) SL_light_cast_obj (obj_index, mask)

Fonction devant être appelée localement à partir d'un objet light , et permettant d'attribuer à ce dernier des objets à ombrer. Function to be called locally from an object light, and for assigning to the latter objects to be shaded. En d'autres termes, elle permet de déterminer pour quels objets la lumière doit projeter une ombre. In other words, it determines which objects to the light should cast shadows.

Arguments: arguments:

obj_index : Index de l'objet devant occluder la lumière, formant ainsi une ombre. obj_index: Index of the object to occluder light, forming a shadow.

mask : Index de la sprite devant être utilisée pour projeter l'ombre, à la place de la sprite de l'objet. mask: Index of the sprite to be used to project the shadow instead of the sprite of the object. Pour ne pas utiliser de mask et donc faire en sorte que la sprite projetée soit celle de l'objet elle-même, indiquez simplement -1. To not use mask and thus ensure that the projected sprite is that of the object itself, please enter -1.

Exemple: Example:

SL_light_cast_obj(obj_character,-1); SL_light_cast_obj (obj_character, -1);

SL_global_cast_obj(layer,obj_index,spr_index,sun,amb) SL_global_cast_obj (layer, obj_index, spr_index, sun, amb)

Fonction devant être appelée localement à partir de l'objet engine , et permettant de définir les objets devant bénéficier d'une ombre globale. Function to be called locally from the engine object, and to define the objects to benefit from a global shadow. Le terme "ombres globales" désigne les ombres solaires et les ombres ambiantes, soit des ombres affectant l'ensemble des objets à l'écran. The term "global shadow" refers solar shadows and ambient shadows or shadows that affect all objects on the screen. Les arguments sun et amb permettent de sélectionner les types de rendus d'ombres globales à effectuer parmi les trois disponibles: The sun and amb arguments to select the types of reports of comprehensive shadows to perform from the three available:

- Ombre solaire projetée ( sun = 1 ): l'objet projette une ombre provenant du soleil, calculée automatiquement à partir de sa sprite. - Projected Shadow solar (sun = 1): the object casts a shadow from the sun, automatically calculated from the sprite. Si vous désirez utiliser une autre sprite que celle de l'objet, vous pouvez définir un mask avec l'argument spr_index . If you wish to use a different sprite than the object, you can define a mask with spr_index argument. Cette option est préférable pour des objets tels que des murs, caisses, barils, véhicules etc. This option is best for objects such as walls, boxes, barrels, vehicles etc.

- Ombre solaire projetée à partir d'une texture ( sun = 2 ): l'objet projette une ombre provenant du soleil, calculée à partir d'une texture prédéfinie et affichée à partir de l'origine de l'objet. - Solar Shade projected from a texture (sun = 2): the object casts a shadow from the sun, calculated from a predefined and displayed texture from the origin of the object. La texture en question doit être définie avec l'argument spr_index . The texture in question must be defined with spr_index argument. Cette option est préférable pour des objets tels que des arbres ou des buissons. This option is best for objects such as trees or bushes.

- Ombre ambiante ( amb = 1 ): l'objet projette une ombre douce résultante de l'illumination globale. - Ambient Shadow (amb = 1): the object casts a shadow resulting soft global illumination.

Arguments: arguments:

layer : Index du layer dans lequel l'ombre de l'objet doit être affichée. layer: the layer index in which the shadow of the object to be displayed. Le système de layers permet de donner un effet de volume au rendu des ombres en faisant en sorte que les objets d'un layer donné (par exemple le layer 2) projettent leurs ombres sur les objets des layers inférieurs (dans notre exemple, les layers 1 et 0). The layers of system gives a volume effect in rendering shadows ensuring that objects of a given layer (eg layer 2) cast their shadows on objects on lower layers (in our example, the layers 1 and 0). Plus nombreux sont les layers, plus nombreuses sont les surfaces utilisées. More numerous are the layers, the more numerous are the surfaces used.

obj_index : Index de l'objet devant projeter une ombre globale. obj_index: Index of the object to project an overall shade.

spr_index : Cette variable joue un rôle différent selon le type de rendu d'ombre solaire utilisé. spr_index: This variable has a different role depending on the type of rendering used solar shade. Pour sun = 1, elle désigne le mask à utiliser pour la projection dynamique de l'ombre (indiquez -1 pour ne pas utiliser de mask). For sun = 1, it means the mask to be used for dynamic projection of the shadow (-1 to indicate not to use a mask). Pour sun = 2, elle désigne l'index de la texture devant être utilisée pour la projection de l'ombre texturée. For sun = 2, it refers to the index of the texture to be used for the projection of the textured shade.

sun : Détermine le type de rendu d'ombre solaire à effectuer. sun: Determines the rendering solar shade to perform. Valeurs: 1 ombre projetée, 2 ombre texturée, 0 désactivé. Values: 1 shadow, shadow textured 2, 0 disabled.

amb : Active/désactive le rendu de l'ombre ambiante. amb: Enable / disable the rendering of the ambient shadows. Valeur booléenne (true / false). Boolean (true / false). Assurez-vous d'avoir généré une ambient map pour la ou les sprite(s) de votre objet avec la fonction SL_sprite_set_ambient(spr_index) . Make sure you have generated a map for ambient or sprite (s) of your object with SL_sprite_set_ambient function (spr_index).

Exemples: Examples:

SL_global_cast_obj(2,obj_tree,spr_tree_shadow,2,1); SL_global_cast_obj (2, obj_tree, spr_tree_shadow, 2.1); // Désigne l'objet obj_tree comme devant projeter une ombre solaire texturée et une ombre ambiante dans le layer 2 // Designates the object as obj_tree to project a shadow textured solar and ambient shadow in the layer 2

SL_global_cast_obj(1,obj_character,-1,1,1); SL_global_cast_obj (1, obj_character, -1,1,1); // Désigne l'objet obj_character comme devant projeter une ombre solaire et une ombre ambiante dans le layer 1 // Designates the object as obj_character to project a solar shade and ambient shadow in the layer 1

SL_cast_sprite(layer,castsun,castamb,castlights,sprite,subimg,x,y,z,xscale,yxscale,rot,alpha) SL_cast_sprite (layer, castsun, castamb, castlights, sprite, subimg, x, y, z, xscale, yxscale, rot, alpha)

Affiche une ombre à partir d'une simple sprite, contrairement aux fonctions précédentes qui projettent des ombres à partir d'objets. Displays a shadow from a single sprite, unlike previous functions that cast shadows from objects. Elle peut être appelée à partir de n'importe quel objet, et doit être exécutée à chaque step à la manière d'une fonction draw native; It can be called from any object, and should be performed in every step on how to draw a native function; elle n'affiche cependant pas la sprite en elle-même, mais uniquement les ombres. it does not display the sprite itself, but only shadows.

Le moment d'exécution de cette fonction doit être configuré avec précaution. The time of execution of this function must be configured carefully. Si son exécution se produit après celle du script SL_engine_render de l'objet engine , la sprite censée être affichée au step courant ne sera pas prise en compte dans le rendu, ce qui créera un effet de décalage. If the execution occurs after that of SL_engine_render the script engine object, the sprite supposed to be displayed at the current step will not be taken into account in the report, creating a lag effect. Assurez-vous donc que cette fonction soit exécutée avant que le rendu soit effectué (dans un événement Step ou Begin Step par exemple). So make sure that this function is executed before the report is made ​​(in a Step or Begin Step event example).

Arguments: arguments:

layer : Index du layer dans lequel les ombres doivent être affichées. layer: the layer index in which the shadow should be displayed. Ne concerne que l'affichage des ombres solaires et ambiantes; Only concerns the display of solar and ambient shadows; vous pouvez y inscrire -1 si vous ne comptez afficher des ombres qu'avec les lumières dynamiques. you can register -1 if you will see shadows with dynamic lights.

castsun : Détermine si une ombre solaire doit être projetée. castsun: Determines if a solar shade should be projected. Valeur booléenne (true / false). Boolean (true / false).

castamb : Détermine si une ombre ambiante doit être affichée. castamb: Determines whether an ambient shadow must be displayed. Valeur booléenne (true / false). Boolean (true / false).

castlights : Détermine si des ombres provenant des lumières dynamiques doivent être projetées. castlights: Determines whether shadows from dynamic lights to be projected. Chaque source lumineuse projettera alors une ombre pour cette sprite. Each light source then project a shadow for this sprite. Valeur booléenne (true / false). Boolean (true / false).

z : altitude de la sprite par rapport au sol. z: elevation of sprite from the ground. Cette valeur n'affecte que le rendu des ombres solaires, et agit par multiplication de la valeur de longueur des ombres. This value affects only rendering solar shadows, and acts by multiplication of the value of length of shadows.

Les arguments restants sont identiques à ceux de la fonction native draw_sprite_ext . The remaining arguments are identical to those of the native function draw_sprite_ext.

Exemple: Example:

SL_cast_sprite(0,1,1,1,spr_object,0,42,42,0,1,1,0,1); SL_cast_sprite (0,1,1,1, spr_object, 0,42,42,0,1,1,0,1);

Affiche une sprite dans le buffer de lumière. Displays a sprite in the light buffer. Unlike a conventional draw function, it can be called in any event (Step for example), and not necessarily Draw. Displays a sprite in the shadow buffer (this is similar to SL_draw_sprite_light) She uses almost the same way as the native function draw_sprite_ext, with some differences: it can be called in any event (Step for example), and it has no color argument. En pratique, cette fonction peut être utile pour ombrer certaines zones de votre map sans pour autant utiliser d'objets (dans le cas par exemple de l'intérieur d'un bâtiment).
