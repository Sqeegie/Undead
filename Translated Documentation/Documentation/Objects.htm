- L'objet engine - The purpose engine

S'il ne vous faut créer qu'un seul objet afin de permettre le fonctionnement de SmartLight , c'est celui-là: il centralise la majeure partie des variables du système, et se charge de l'ensemble du rendu et de l'affichage. If you do create one object to allow the operation of SmartLight is one: it centralizes most of the variables in the system, and is responsible for the entire issued and the display. Il gère le cycle jour/nuit, ainsi que les systèmes d'ombres solaires et d'ombres ambiantes (les variables relatives aux lumières dynamiques, quant à elles, sont gérées localement par les objets light eux-mêmes). It manages the day / night cycle and the solar shading systems and ambient shadows (variables related to dynamic lights, in turn, are managed locally by light objects themselves). Assurez-vous que la variable depth (profondeur) de l'objet engine soit inférieure à celle des objets sur lesquels les effets d'ombres et lumières doivent être affichés. Make sure the variable depth (depth) of the engine object is less than that of the objects on which the light and shadow effects to be displayed.

Events are structured as follows: initialization and rendering and display, and destruction if necessary. - L'objet light - The object light

Le ou les objet(s) light (vous pouvez en créer plusieurs) permet de gérer les lumières dynamiques. The object (s) light (you can create more) manages dynamic lights. Le rendu des ombres est totalement automatisé: il suffit de configurer quelques variables dans l'objet light lui-même, et SmartLight se charge du reste. Rendering shadows is fully automated: just set some variables in the object light itself, and SmartLight does the rest. En soi cet objet ne fait rien de plus que gérer des variables: le rendu de la lumière, lui, est effectué par l'objet engine . In itself this object is nothing more than manage the variables: the rendering of light, it is done by the engine object. Il est donc techniquement possible de gérer ces variables à partir de n'importe quel objet, sans forcément user d'un objet light dédié. It is technically possible to manage these variables from any object, not necessarily use a dedicated light object.

Contrairement à l'objet engine , il n'est pas obligatoire de créer un objet light pour que le système puisse fonctionner. Unlike the engine object, it is not mandatory to create a light object for the system to function. Par exemple, vous pouvez très bien faire usage uniquement du cycle jour/nuit et rien d'autre: aucun bug ne naîtra de l'absence d'objet light . For example, you can very well make use only of the day / night cycle and nothing else: no bugs will arise from the lack of light object.

Les événements sont structurés très simplement: initialisation, et destruction si nécessaire. The events are structured very simply: initialization, and destruction if necessary. Rien de plus. Nothing more.

Note 1: In the GSM version of the engine, the system does not automatically recognize the light objects: they therefore you need to declare them manually with SL_add_light function (object).
