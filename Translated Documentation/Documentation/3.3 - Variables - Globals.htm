3.3 - Variables - Globales 3.3 - Variables - Global

Le système de SmartLight comprend quelques variables globales (dont la grande majorité est faite de variables automatiques), permettant aux différents objets de communiquer entre eux indépendamment des noms leur étant attribués par l'utilisateur. The SmartLight system includes some global variables (which the vast majority is made ​​of automatic variables), allowing different objects communicate with each other regardless of their names being assigned by the user.

global.sl_viewid global.sl_viewid

Index de la view dans laquelle les effets de lumière doivent être affichés. Index of the view in which light effects to be displayed. Dans un souci d'ergonomie, il est préférable de la déclarer dans l'objet engine lors de l'initialisation de celui-ci. For the sake of ergonomics, it is best to declare in the engine object during initialization of it.

Exemple: Example:

global.sl_viewid=view_current; global.sl_viewid = view_current;

global.sl_z[obj_id] global.sl_z [obj_id]

Variable z de l'objet d'id obj_id , déterminant le facteur d'altitude par rapport au sol de l'ensemble des instances issues de cet objet. Variable z of the object id obj_id determining the altitude factor from the ground of all instances from this object. Elle agit sur le rendu des ombres solaires, en donnant l'impression que les objets sont élevés au dessus du sol. It acts on the rendering of solar shadows, giving the impression that the objects are high above the ground. Elle est déclarée automatiquement avec une valeur par défaut de 0. It is automatically declared with a default value of 0.

global.sl_time global.sl_time

Heure courante. Current time. Elle est supérieure ou égale à 0 et inférieure à 24. Sa vitesse de variation est déterminée par la variable sl_timespeed de l'objet engine . It is greater than or equal to 0 and less than 24. The speed of variation is determined by the variable of the engine sl_timespeed object.

Exemples: Examples:

Pour sl_time=0, il est minuit. For sl_time = 0, it is midnight.

Pour sl_time=4, il est 4 heures du matin. For sl_time = 4, it is 4 am.

Pour sl_time=15, il est 15 heures (ou 3 heures de l'après-midi). For sl_time = 15, it is 15 hours (or 3 in the afternoon).

Pour sl_time=19.5, il est 19h 30 (ou 7h 30 de l'après-midi). For sl_time = 19.5, it is 19h 30 (or 7 am to 30 in the afternoon).

Le format décimal de la variable global.sl_time comprend donc les minutes, mais pas sous leur forme directe. The decimal format global.sl_time variable therefore includes the minutes but not under their direct form. Voici un petit code permettant d'obtenir l'heure et les minutes à partir de la variable global.sl_time brute: Here's a little code to get the hour and minutes from the gross global.sl_time variable:

heure=global.sl_time-frac(global.sl_time); global.sl_time-hour = frac (global.sl_time);

minutes=floor(60*frac(global.sl_time)); minutes = floor (60 * frac (global.sl_time));

Cette variable est en lecture seule et ne peut être modifiée directement. This variable is read-only and can not be changed directly. Cependant, la fonction SL_set_time(time) permet de modifier l'heure à votre guise. However, SL_set_time function (time) lets you change the time on your own.

id d'une surface servant de buffer global pour le rendu des lumières. id of a surface serving as a global buffer for rendering lights.
